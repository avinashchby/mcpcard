# mcpcard

Generate MCP Server Cards per SEP-1649 — let clients discover your server without connecting.

## Quick Start

```bash
npx @avinashchby/mcpcard scan ./src
```

## What It Does

MCP clients currently have no way to know what tools, resources, or prompts a server exposes without establishing a live connection first. `mcpcard` generates a static `server-card.json` file placed at `/.well-known/mcp/server-card.json` that describes your server's full capabilities — tools, resources, prompts, transport, and authentication — in a machine-readable format per the SEP-1649 specification. Cards can be generated by statically scanning TypeScript, JavaScript, or Python source files, or by introspecting a running server directly over stdio or HTTP. Once generated, the card can be validated against the SEP-1649 JSON Schema and served over HTTP so registries, crawlers, and MCP clients can discover your server without making a connection.

## Features

- Static source scanning for TypeScript/JavaScript (via TypeScript compiler AST) and Python (via regex on FastMCP patterns)
- Live introspection of running servers via stdio or HTTP using the MCP protocol
- JSON Schema validation against the SEP-1649 spec (exit 0 = valid, exit 1 = errors printed)
- HTTP server that serves the card at `/.well-known/mcp/server-card.json`
- Express middleware (`mcpCardMiddleware`) for embedding the card endpoint in your own server
- Deduplicates and alphabetically sorts tools, resources, and prompts
- Reads `package.json` for server name, version, and description metadata as fallbacks
- Usable as a Node.js library alongside the CLI

## Usage

Scan a TypeScript/JavaScript or Python source directory and write the card to the default path:

```bash
npx @avinashchby/mcpcard scan ./src
# Writes: .well-known/mcp/server-card.json
```

Scan with a custom output path and verbose logging:

```bash
npx @avinashchby/mcpcard scan ./src --output ./public/server-card.json --verbose
```

Introspect a running stdio server:

```bash
npx @avinashchby/mcpcard live --command "node dist/server.js"
```

Introspect a running HTTP server:

```bash
npx @avinashchby/mcpcard live --url http://localhost:8080
```

Validate an existing card file:

```bash
npx @avinashchby/mcpcard validate .well-known/mcp/server-card.json
```

Serve a card over HTTP (validates first, refuses to serve if invalid):

```bash
npx @avinashchby/mcpcard serve .well-known/mcp/server-card.json --port 3000
# Card available at http://localhost:3000/.well-known/mcp/server-card.json
```

## Example Output

```json
{
  "name": "my-mcp-server",
  "version": "1.2.0",
  "description": "An MCP server that does useful things.",
  "capabilities": {
    "tools": true,
    "resources": true,
    "prompts": false
  },
  "transport": {
    "type": "stdio"
  },
  "tools": [
    {
      "name": "fetch_url",
      "description": "Fetch the contents of a URL"
    },
    {
      "name": "run_query",
      "description": "Execute a SQL query against the database"
    }
  ],
  "resources": [
    {
      "uri": "db://schema",
      "name": "get_schema",
      "description": "Returns the database schema"
    }
  ],
  "protocol_version": "2025-11-05"
}
```
